# E-commerce Sales Data Analysis

## Project Overview

This project analyzes sales data from an e-commerce platform to identify key business insights reated to revenue trends, product performance, and geographic distribution.

The goal of this analysis is to simulate a real-world Data Analysis workflow, transforming raw transactional data into actionable insights that could support business decision-making

## Business questions

This project explores the following questions:
- How has revenue evolved over time?
- Which product categories generate the most revenue?
- Which product sell the most frequently?
- Where are the pltform's customers located?

## Dataset

The dataset contains multiple tables representing different aspects of an e-commerce marketplace, including:
- Orders
- Order items
- Customers
- Products
- Product category traslation

This tables were combined through data joins to create a unified dataset for analysis.

## Project Workflow

The analysis followed several steps:

1. **Data Loading:**
   Importing raw datasets and inspecting their structure.
2. **Data Cleaning:**
    Handling missing values and filtering delivered orders.
3. **Data Integration:**
    Merging multiple datasets to create a complet6e sales table.
4. **Exploratory Data Analyis:**
   - Monthly revenue trends
   - Revenue by product category
   - Most sold products
   - Customer geographic distribution
5. **Visualization:**
   Creating charts to communicate insights clearly.

## Key Insights

- **Revenue Trends:** Sales show steady growth over time.

- **Top Categories:** Health & Beauty, Watches & Gifts, and Bed & Bath generate the highest revenue.

- **Product Demand:** Certain products appear frequently across orders, indicating strong demand.

- **Geographic Concentration:** Customer activity is concentrated in a small number of states, with São Paulo representing the largest market.

These findings highlight how product demand and geographic concentration drive overall marketplace performance.
